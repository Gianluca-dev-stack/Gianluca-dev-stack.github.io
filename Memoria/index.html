Approve once. Reuse safely.

Memoria keeps the beta loop small: create a proposal, review it, approve or reject it, then preview what approved memory would return for a project-scoped request.

Step 1

Create a proposal

New project context starts pending, not active. Suggested memory does not reach AI tools before review.

Step 2

Approve or reject

Edit useful context before approval, or reject it. Pending, rejected, archived, deprecated, and replaced memories are not shared.

Step 3

Preview retrieval

Context Preview shows approved-only project memory with score, snippet, matched terms, tags, source, and project path.

Skeptical developer FAQ.

Short answers for the objections that matter before a tester gives Memoria five minutes.

Is this just RAG?

No. RAG retrieves facts from documents. Memoria focuses on approved project context that should influence future AI-tool use.

Does Memoria automatically learn my repo?

No. The private beta is approval-first. Pending memories stay inactive until approved.

Can an AI tool approve its own memory?

No. Governed MCP tools can create pending proposals and retrieve approved context, but approval stays in Memoria.

What is a receipt?

A receipt is content-free metadata for what Memoria would share or suppress. It does not store raw prompts, queries, source code, paths, or tool payloads.

Where is data stored?

The private beta stores data locally by default in SQLite. The tester database is usually inside the tester folder.

Do testers receive source code?

No. Private beta testers receive a binary package. The binary is a beta boundary, not DRM.
